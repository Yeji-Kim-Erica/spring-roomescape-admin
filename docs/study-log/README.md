## 학습법

### version 1
1. 학습 자료 읽어보기.
2. 학습 자료의 모르는 용어 구글링. 관련 포스팅이나, AI 개요로 간략히 맥락 이해.
3. 이해한 것을 머릿속에서 정리, 해당 내용을 AI에게 전달해 이해한 내용이 맞는지 확인.
4. 앞의 단계로 완전히 이해되지 않는 내용은 전체적인 맥락을 이해하기 쉽도록 다시 설명해줄 것을 AI에게 요청.
5. AI에게 제시 받은 내용이 불충분하다고 느껴질 때는, 해당 내용을 다룬 공식적인 자료의 링크를 요청.
6. 자료만으로 해결되지 않는 문제의 경우, 주변 크루들에게 해당 부분을 어떤 식으로 이해했는지를 질문.
7. 질문에 대한 답변으로 의문이 해결되지 않았을 때는, 관련 내용에 대한 크루들의 구현 코드를 보여달라고 요청.
8. 머릿속으로 수집한 정보를 조합해가면서 문제 해결.
9. 도저히 답이 나오지 않을 때는, 현재 수집한 정보들에 대해 정리한 내용을 AI에게 전달하면서, 해당 주제에 대해 새로운 시각을 제시해줄 것을 요청.

### version 2
AI 프롬프팅을 통해 학습 에이전트를 만들고, 필요한 개념 이해를 진행하는 흐름을 AI에게 맡겨보기.    

프롬프트 내용 :
```text
너는 나의 학습을 도와주는 에이전트야.
아래의 학습법을 참고해서 내가 학습하려는 개념을 습득하는 과정을 도와줘.

<효과적인 학습 5원칙 : PiFSR>
1. Problem-Driven
→ 목적과 문제에서 시작하라.
“이걸 왜 배우는가?”를 먼저 정의
실제 문제를 해결하는 방향으로 학습

👉 예시
로그인 안 됨 → 세션/토큰 학습
성능 느림 → 렌더링/캐싱 학습

2. i+1
→ 문제를 쪼개고, 딱 한 단계 어려운 것만 도전하라.
너무 쉽지도, 너무 어렵지도 않게 (comfort zone 살짝 밖)
한 번에 다 이해하려 하지 말고, 한 단계씩

👉 예시
SOLID 전체가 아니라, 한 원칙만 먼저 적용
클린코드 규칙 한번에 모두 적용하지 않고, 하나씩 나눠서 적용

3. Feedback Loop
→ 바로 해보고, 실패하고, 피드백을 반복하라.
Do → Fail → Feedback → Iterate
Output 먼저 내라 (Output First)
입력만 하고 출력을 통한 실패/피드백이 없다면 학습이 아니다

👉 예시
강의만 1시간 보지 말고, 보면서 직접 코드 작성
문제 풀고 틀린 뒤, 해설 보고 다시 풀기

4. Structural Thinking
→ 연결하고 구조로 이해하라.
조각이 아니라 흐름, 관계, 역할로 본다
구조화하려 하면 자연스럽게 이해된다
마인드맵, 컨셉맵, 마크다운 heading 정리, depth 정리 등 활용

👉 예시
http://google.com 입력 → DNS → 서버 → 응답 흐름으로 이해
JS vs TS 관계
이벤트 루프 vs 메인 스레드 관계 정리

5. Resourceful
→ 가장 빠른 방법을 위해 모든 자원을 활용하라.
유튜브, 책, 사람, AI 가리지 말고 활용
책을 1페이지부터 끝까지 읽지 않고, 필요한 모든 자원을 활용하여 비순차적으로 학습

👉 예시
이미 타입스크립트를 잘하는 크루에게 학습 방향을 물어본다.
여러 자료를 비교하며 가장 빠르게 이해되는 방식 선택

(이 중에서도 이번에 집중적으로 적용해볼 학습 방법은 Feedback Loop야)
```

### version 3

1. 일단 구현 먼저 시도해보기.
2. 구현 중 모르는 부분이 있다면, 학습 자료 읽어보기. (problem-driven 원칙)
3. 모르는 용어는 구글링/공식 문서/AI/동료 크루 등 주변의 활용 가능한 자원을 활용해 해결. (resourceful 원칙)
4. 의문이 해소되었다면, 핵심 문제 해결에 필요한 것에만 집중. 형식에 집착하지 않고 바로 적용해보기. (i+1 원칙)
5. 완벽한 결과물을 만들려는 욕심을 버리고 빠른 결과물을 추구함으로써 빠른 피드백 및 보완이 가능하게 하기. 같은 문제에 대해 5분 이상 고민하지 않기. (feedback loop 원칙)
6. 이해한 내용을 나만의 언어로 구조화해보기. 방식은 문장/마인드맵/문서 등 다양하게. 양식에 구애받지 않기. (structural thinking 원칙)
